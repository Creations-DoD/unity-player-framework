IEventBus
PlayerFramework.Core.Contracts

Typed, struct-based publish/subscribe event bus scoped to a single player entity. All cross-subsystem communication for state notifications flows through here. Events must be value types (structs) to avoid heap allocation. The event bus is registered with the IServiceLocator and available to all subsystems via GetService<IEventBus>().

Methods

Registers a handler to receive events of type T. The handler will be called synchronously during Publish<T>(). Subscribe during Initialize() or Enable() — never during a Tick() call.
Parameters
TypeNameDescription

Returns
void
Preconditions

Throws

void Unsubscribe<T>(Action<T> handler) where T : struct

void

IInputProvider
PlayerFramework.Core.Contracts

The critical multiplayer seam. Abstracts the source of player input so that the rest of the framework is completely agnostic to whether a human, AI, network authority, or replay system is driving the player. Nothing in the framework except HumanInputProvider reads from UnityEngine.InputSystem. Implementations: HumanInputProvider, AIInputProvider, NetworkInputProvider, NullInputProvider.

Methods

PlayerInputFrame GetCurrentFrame()
Returns the input snapshot for the current simulation tick. Called once per Tick() by InputSystem and the result is cached as InputSystem.CurrentFrame for all other subsystems to read. Must be allocation-free.
Returns
PlayerInputFrame — a complete snapshot of all input state for this tick.
Preconditions
  • The provider has been initialised (for HumanInputProvider: the Input Action Asset has been loaded)
